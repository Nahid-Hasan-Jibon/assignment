Discuss the scope of var, let, and const

To declare variable, we use var,let and const in JavaScript. But their scopes are different. Scope means the accessiblity of variable.

well, let us know about scopes of var, let and const

var

Var is globally scopped. This means we can access those variables which are declared with var from anywhere . The variables we declare with var are hoisted to the top of their containg function. Var is also function scoped which means the variable declared with var are accessible within the function in which they are defined. the value of var can be changed by declaring another var with the same name

let

When we use let to declare variables they become block scoped. This means we can access the value of the variable only within the block where they are defined . May be the block is a loop or a function or a conditional statement. So we can say, The scope of a let variable is only block scoped. It can't be accessible outside the particular block . Usually let variable has no hoisting issues .

const

const is also block scoped like let .That means it is also accessible only within the block where it is defined. The variable declared with const can not be reassigned any new value . the value is always constant here.

In modern Java Script we dont usually use var any more. Instead of 'var' we use 'let' and 'const'

Tell us the use cases of null and undefined

Null

Null is basically an assignment value. The variable which has been assigned as null contains no value and is empty.

Use case of null :

  • If there is no meaningfull value present in a variable we can assign 'null' to that variable.
  • Null is often used when we want to clear a variable or indicate that it hasn't been assigned a value yet
  • We can use 'null' as a placeholder until we have the actual value to assign
  • When a function doesn't have a valid return . Function can return 'null' to indicate that there is no meaningfull result or value

    • Undefined

    Undefined is when we declare a variable but do not assign a value to define it, the variable becomes undefined.

    Use case of Undefined :

  • The undefined property indicates that a variable has not been assigned a value, or not declared at all.
  • At the time of calling a function if we dont provide a parameter. The function defaults to 'undefined' . That means 'undefined' can be used to check if value is passed or not
  • If any object properties haven't been set , they are undefined . We can check if a specific property exists on a object
